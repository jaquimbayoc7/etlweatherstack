🔄 Proceso ETL Completo

Aprende a crear un pipeline de extracción, transformación y carga de datos usando Weatherstack API, Python y GitHub

Dominarás las 4 fases esenciales: configuración del entorno, generación de credenciales, desarrollo del proceso ETL y publicación en repositorios

Fase 1: Alistamiento de WSL Ubuntu 24 en Windows

Windows Subsystem for Linux (WSL) te permite ejecutar un entorno Linux completo en Windows. Es perfecto para desarrollo de data pipelines.

⚠️ Requisitos Previos

  • Windows 10 (build 2004 o superior) o Windows 11
  • Acceso a PowerShell como administrador
  • Al menos 5 GB de espacio libre
  • Virtualización habilitada en BIOS

1Habilitar WSL

Abre PowerShell como administrador y ejecuta:

wsl --install

Esto instala WSL 2 y Ubuntu 24 automáticamente. Si no especificas distribución, Ubuntu es la default.

2Reinicia el Sistema

Después de ejecutar el comando, reinicia tu computadora. Esto es crítico para que los cambios del kernel tomen efecto.

3Configura tu Usuario en Ubuntu

Cuando reinicies, WSL se abrirá automáticamente. Te pedirá crear un usuario. Completa:

Enter new UNIX username: desarrollo
Enter new UNIX password: tu_password_segura
Retype new UNIX password: tu_password_segura

4Actualiza Ubuntu

Una vez dentro de Ubuntu, actualiza los paquetes:

sudo apt update
sudo apt upgrade -y

5Instala Python 3.11+ y pip

Verifica qué versión de Python viene por defecto:

python3 --version

Si es menor a 3.11, instala la más reciente:

sudo apt install -y python3 python3-pip python3-venv
pip3 --version

6Instala Git y Configúralo

Git es esencial para versionar tu código:

sudo apt install -y git
git config --global user.name "Tu Nombre"
git config --global user.email "tu.email@example.com"

7Instala Visual Studio Code con WSL Remote

En Windows (fuera de WSL), descarga VS Code desde https://code.visualstudio.com/

Luego instala la extensión "Remote - WSL" desde el Marketplace de VS Code.

# Desde PowerShell (Windows):
code .

8Verifica la Instalación

Desde PowerShell, abre WSL y verifica:

# En PowerShell
wsl

# Ya estás en Ubuntu:
python3 --version
git --version
pip3 --version

✅ ¡WSL Listo!

Ahora tienes un entorno Linux completo en Windows con Python, Git y herramientas de desarrollo. Puedes trabajar directamente desde VS Code con la extensión Remote - WSL.

5Crea el Script Principal de Extracción

Crea scripts/extractor.py:

#!/usr/bin/env python3
import os
import requests
import json
import pandas as pd
from datetime import datetime
from dotenv import load_dotenv
import logging

# Cargar variables de entorno
load_dotenv()

# Configurar logging
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(levelname)s - %(message)s',
    handlers=[
        logging.FileHandler('logs/etl.log'),
        logging.StreamHandler()
    ]
)
logger = logging.getLogger(__name__)

class WeatherstackExtractor:
    def __init__(self):
        self.api_key = os.getenv('API_KEY')
        self.base_url = os.getenv('WEATHERSTACK_BASE_URL')
        self.ciudades = os.getenv('CIUDADES').split(',')
        
        if not self.api_key:
            raise ValueError("API_KEY no configurada en .env")
    
    def extraer_clima(self, ciudad):
        """Extrae datos de clima para una ciudad específica"""
        try:
            url = f"{self.base_url}/current"
            params = {
                'access_key': self.api_key,
                'query': ciudad.strip()
            }
            
            response = requests.get(url, params=params, timeout=10)
            response.raise_for_status()
            
            data = response.json()
            
            if 'error' in data:
                logger.error(f"❌ Error en API para {ciudad}: {data['error']['info']}")
                return None
            
            logger.info(f"✅ Datos extraídos para {ciudad}")
            return data
            
        except Exception as e:
            logger.error(f"❌ Error extrayendo datos para {ciudad}: {str(e)}")
            return None
    
    def procesar_respuesta(self, response_data):
        """Procesa la respuesta JSON a formato estructurado"""
        try:
            current = response_data.get('current', {})
            location = response_data.get('location', {})
            
            return {
                'ciudad': location.get('name'),
                'pais': location.get('country'),
                'latitud': location.get('lat'),
                'longitud': location.get('lon'),
                'temperatura': current.get('temperature'),
                'sensacion_termica': current.get('feelslike'),
                'humedad': current.get('humidity'),
                'velocidad_viento': current.get('wind_speed'),
                'descripcion': current.get('weather_descriptions', ['N/A'])[0],
                'fecha_extraccion': datetime.now().isoformat(),
                'codigo_tiempo': current.get('weather_code')
            }
        except Exception as e:
            logger.error(f"Error procesando respuesta: {str(e)}")
            return None
    
    def ejecutar_extraccion(self):
        """Ejecuta la extracción para todas las ciudades"""
        datos_extraidos = []
        
        logger.info(f"Iniciando extracción para {len(self.ciudades)} ciudades...")
        
        for ciudad in self.ciudades:
            response = self.extraer_clima(ciudad)
            if response:
                datos_procesados = self.procesar_respuesta(response)
                if datos_procesados:
                    datos_extraidos.append(datos_procesados)
        
        return datos_extraidos

if __name__ == "__main__":
    try:
        extractor = WeatherstackExtractor()
        datos = extractor.ejecutar_extraccion()
        
        # Guardar como JSON
        with open('data/clima_raw.json', 'w') as f:
            json.dump(datos, f, indent=2)
        logger.info(f"📁 Datos guardados en data/clima_raw.json")
        
        # Guardar como CSV
        df = pd.DataFrame(datos)
        df.to_csv('data/clima.csv', index=False)
        logger.info(f"📁 Datos guardados en data/clima.csv")
        
        print("\n" + "="*50)
        print("RESUMEN DE EXTRACCIÓN")
        print("="*50)
        print(df.to_string())
        print("="*50)
        
    except Exception as e:
        logger.error(f"Error en extracción: {str(e)}")

6Crea Script de Visualización

Crea scripts/visualizador.py para ver los datos graficados:

#!/usr/bin/env python3
import pandas as pd
import matplotlib.pyplot as plt
import numpy as np

# Cargar datos
df = pd.read_csv('data/clima.csv')

# Crear figura con múltiples gráficas
fig, axes = plt.subplots(2, 2, figsize=(15, 10))
fig.suptitle('Análisis de Clima por Ciudades', fontsize=16, fontweight='bold')

# Gráfica 1: Temperaturas
ax1 = axes[0, 0]
ax1.bar(df['ciudad'], df['temperatura'], color='#ff6b6b')
ax1.set_title('Temperatura Actual (°C)')
ax1.set_ylabel('Temperatura (°C)')
ax1.tick_params(axis='x', rotation=45)
ax1.grid(axis='y', alpha=0.3)

# Gráfica 2: Humedad
ax2 = axes[0, 1]
ax2.bar(df['ciudad'], df['humedad'], color='#4ecdc4')
ax2.set_title('Humedad Relativa (%)')
ax2.set_ylabel('Humedad (%)')
ax2.tick_params(axis='x', rotation=45)
ax2.grid(axis='y', alpha=0.3)

# Gráfica 3: Velocidad del Viento
ax3 = axes[1, 0]
ax3.scatter(df['ciudad'], df['velocidad_viento'], s=200, color='#95e1d3')
ax3.set_title('Velocidad del Viento (km/h)')
ax3.set_ylabel('Velocidad (km/h)')
ax3.tick_params(axis='x', rotation=45)
ax3.grid(alpha=0.3)

# Gráfica 4: Sensación Térmica vs Temperatura
ax4 = axes[1, 1]
x = np.arange(len(df))
width = 0.35
ax4.bar(x - width/2, df['temperatura'], width, label='Temperatura', color='#ff6b6b')
ax4.bar(x + width/2, df['sensacion_termica'], width, label='Sensación Térmica', color='#ffa07a')
ax4.set_title('Temperatura vs Sensación Térmica')
ax4.set_ylabel('Temperatura (°C)')
ax4.set_xticks(x)
ax4.set_xticklabels(df['ciudad'], rotation=45)
ax4.legend()
ax4.grid(axis='y', alpha=0.3)

plt.tight_layout()
plt.savefig('data/clima_analysis.png', dpi=300, bbox_inches='tight')
logger.info("✅ Gráficas guardadas en data/clima_analysis.png")
plt.show()

import schedule
import time

def ejecutar_etl():
    extractor = WeatherstackExtractor()

schedule.every(1).hours.do(ejecutar_etl)

    • 📋 Formato Recomendado para la Socialización (16 Parejas - 1 Pregunta c/u)

    Paso 1: Forma las 16 parejas y asigna a cada pareja UNA pregunta única del listado anterior

    Paso 2: Cada pareja discute su pregunta asignada durante 2-3 minutos y prepara una respuesta conjunta concisa

    Paso 3: Todas las parejas presentan simultáneamente o en orden (máximo 1-2 minutos por presentación)

    Paso 4: El facilitador (profesor) selecciona 2-3 preguntas clave para profundizar con toda la clase

    Paso 5: Debate abierto donde otros estudiantes pueden complementar respuestas

    💡 Ventaja de esta estructura: Cada pareja es responsable de una pregunta específica, lo que evita repeticiones y asegura cobertura de todos los temas en una sesión corta.

    ✨ Beneficios de esta Reflexión Colaborativa (Una Pregunta por Pareja)

    • Concentración: Cada pareja se enfoca profundamente en Una sola pregunta sin distracciones
    • Equidad: Todas las parejas participan activamente - nadie se queda sin presentar
    • Cobertura Total: En 20-30 minutos se cubren 16 preguntas diferentes sobre todos los temas
    • Debate Enriquecido: Después, el profesor profundiza en las respuestas más interesantes
    • Responsabilidad Compartida: Ambos integrantes de la pareja deben entender su pregunta
    • Perspectivas Múltiples: 16 respuestas diferentes te muestran cómo otros compañeros piensan
