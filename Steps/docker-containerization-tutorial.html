🐳 Containerización con Docker

Empaqueta tu aplicación completa en contenedores para despliegue consistente en cualquier entorno

Desde desarrollo local hasta producción en la nube, garantiza que tu ETL y Dashboard funcionen idénticamente en todos lados

Fase 1: Instalación de Docker

Docker es una plataforma de containerización que permite empaquetar aplicaciones con todas sus dependencias.

¿Qué es Docker?

Docker es un contenedor que aísla tu aplicación (código, dependencias, configuración) en una unidad independiente que funciona igual en desarrollo, testing y producción. Es como una máquina virtual pero mucho más liviana.

1Instala Docker Desktop (Windows/Mac)

Ve a docker.com/products/docker-desktop

  • Descarga Docker Desktop para tu SO
  • Ejecuta el instalador
  • Sigue el asistente de instalación
  • Reinicia tu computadora

2Instalación en Linux

Si estás en Linux o WSL, usa el comando:

sudo apt update
sudo apt install -y docker.io docker-compose

# Dale permisos al usuario actual
sudo usermod -aG docker $USER

# Reinicia sesión o ejecuta:
newgrp docker

3Verifica Instalación

Comprueba que Docker está funcionando:

docker --version
docker run hello-world

# Deberías ver un mensaje de bienvenida de Docker

📦 Conceptos Clave de Docker

  • Imagen: Plantilla con tu aplicación y dependencias
  • Contenedor: Instancia ejecutable de una imagen
  • Dockerfile: Archivo que define cómo construir la imagen
  • Docker Compose: Herramienta para ejecutar múltiples contenedores
  • Registry: Repositorio de imágenes (Docker Hub)

✅ Docker Instalado

Estás listo para containerizar tu aplicación.

Fase 2: Crear Dockerfile

Un Dockerfile es un script que define los pasos para construir tu imagen Docker.

1Crea el Dockerfile

En la raíz de tu proyecto, crea Dockerfile (sin extensión):

# Usar imagen base oficial de Python
FROM python:3.11-slim

# Establecer directorio de trabajo
WORKDIR /app

# Copiar requirements.txt
COPY requirements.txt .

# Instalar dependencias Python
RUN pip install --no-cache-dir -r requirements.txt

# Copiar todo el código del proyecto
COPY . .

# Crear directorios necesarios
RUN mkdir -p logs data

# Exponer puerto 8501 (Streamlit)
EXPOSE 8501

# Variable de entorno para Python
ENV PYTHONUNBUFFERED=1

# Healthcheck
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
    CMD python -c "import sys; sys.exit(0)"

# Comando por defecto: ejecutar Streamlit
CMD ["streamlit", "run", "dashboard_interactive.py", "--server.port=8501", "--server.address=0.0.0.0"]

2Crea .dockerignore

En la raíz del proyecto, crea .dockerignore para excluir archivos innecesarios:

3Construir la Imagen

Desde la raíz del proyecto:

docker build -t etl-weatherstack:1.0 .

# Ver la imagen creada
docker images

Esto tardará un par de minutos mientras descarga la imagen base e instala dependencias.

🔍 Explicación del Dockerfile

  • FROM python:3.11-slim - Imagen base pequeña de Python
  • WORKDIR /app - Directorio de trabajo dentro del contenedor
  • COPY requirements.txt . - Copia dependencias
  • RUN pip install... - Instala paquetes Python
  • COPY . . - Copia todo el código
  • EXPOSE 8501 - Abre puerto para Streamlit
  • CMD - Comando a ejecutar cuando inicia el contenedor

Fase 3: Docker Compose (Multi-contenedor)

Docker Compose permite orquestar múltiples contenedores (app + PostgreSQL) con un solo comando.

1Crea docker-compose.yml

En la raíz del proyecto:

version: '3.8'

services:
  # Servicio PostgreSQL
  postgres:
    image: postgres:15-alpine
    container_name: weatherstack_db
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres_password_segura
      POSTGRES_DB: weatherstack_etl
    ports:
      - "5432:5432"
    volumes:
      # Persiste datos en volumen
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - etl-network

  # Servicio de la aplicación Streamlit
  app:
    build: .
    container_name: weatherstack_app
    environment:
      # Variables de conexión a BD
      DB_HOST: postgres
      DB_PORT: 5432
      DB_USER: postgres
      DB_PASSWORD: postgres_password_segura
      DB_NAME: weatherstack_etl
      API_KEY: ${API_KEY}
      WEATHERSTACK_BASE_URL: https://api.weatherstack.com
      CIUDADES: Bogota,Medellin,Cali,Barranquilla,Cartagena
    ports:
      - "8501:8501"
    depends_on:
      postgres:
        condition: service_healthy
    volumes:
      # Monta código local para desarrollo (hot reload)
      - .:/app
      - /app/__pycache__
    networks:
      - etl-network
    command: streamlit run dashboard_interactive.py --server.port=8501 --server.address=0.0.0.0

# Volúmenes persistentes
volumes:
  postgres_data:
    driver: local

# Red personalizada
networks:
  etl-network:
    driver: bridge

2Crea archivo .env para Docker

Crea .env.docker con variables seguras:

API_KEY=tu_api_key_aqui
DB_HOST=postgres
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=tu_contraseña_muy_segura
DB_NAME=weatherstack_etl

⚠️ Nunca commiteaes este archivo a Git.

3Ejecuta Docker Compose

Construye e inicia todos los servicios:

# Construir imágenes
docker-compose build

# Iniciar servicios en background
docker-compose up -d

# Ver logs
docker-compose logs -f app

# Ver estado de servicios
docker-compose ps

4Accede a tu Dashboard

Tu aplicación estará disponible en:

http://localhost:8501

PostgreSQL también estará corriendo en el puerto 5432 interno.

🔗 Ventajas de Docker Compose

  • ✅ Crea múltiples contenedores con un comando
  • ✅ Red interna entre contenedores
  • ✅ Volúmenes persistentes para BD
  • ✅ Variables de entorno centralizadas
  • ✅ Health checks automáticos
  • ✅ Fácil de reproducir en cualquier máquina

✅ Contenedores Orquestados

Tu aplicación y base de datos funcionan juntas en contenedores aislados.

Fase 4: Build y Run Detallado

Comandos útiles para trabajar con Docker en desarrollo.

1Comandos Docker Básicos

# Ver imágenes disponibles
docker images

# Ver contenedores en ejecución
docker ps

# Ver todos los contenedores (incluyendo detenidos)
docker ps -a

# Detener un contenedor
docker stop nombre_contenedor

# Eliminar un contenedor
docker rm nombre_contenedor

# Ver logs en tiempo real
docker logs -f nombre_contenedor

# Ejecutar comando dentro del contenedor
docker exec -it nombre_contenedor bash

2Comandos Docker Compose

# Construir imágenes
docker-compose build

# Iniciar servicios
docker-compose up

# Iniciar en background (-d = detached)
docker-compose up -d

# Detener servicios
docker-compose down

# Ver logs de un servicio
docker-compose logs -f app

# Ver estado
docker-compose ps

# Ejecutar comando en un contenedor
docker-compose exec app bash

# Reconstruir después de cambios
docker-compose up -d --build

3Debugging en Contenedor

Si algo no funciona, conecta al contenedor:

# Acceder a bash del contenedor app
docker-compose exec app bash

# Una vez dentro:
python scripts/test_db.py  # Prueba conexión BD
python scripts/extractor_db.py  # Ejecuta ETL
ls -la  # Lista archivos
cat logs/etl.log  # Ver logs

# Salir
exit

🐛 Solución de Problemas Comunes

  • Puerto en uso: docker-compose down y luego up
  • BD no conecta: Verifica que postgres esté healthy: docker-compose ps
  • Cambios no se ven: Reconstruye: docker-compose up -d --build
  • Ver logs: docker-compose logs -f nombre_servicio
